# ui-rules.md

## Purpose

Defines UI behavior rules.

---

# Design System

Follow:

- ui-tokens.md
- profile.png
- design references

---

# Theme

Default Theme:

Aerix Light

Light mode is primary.

Dark mode may be added later.

---

# Borders

Use:

1px hairline borders

Color:

#ebebeb

---

# Shadows

Keep minimal.

Avoid heavy shadows.

---

# Radius

Buttons:

6px

Cards:

12px

Large Cards:

16px

Pills:

100px

---

# Icons

Lucide only.

No emojis.

---

# Spacing

Use spacing tokens.

Avoid arbitrary values.

---

# Empty States

Never leave blank screens.

Use empty-states.md.

---

# Mobile

Every feature must work on mobile.

No desktop-only flows.

---

# Analytics

Never show fake data.

Never generate placeholder metrics.

Use real values or empty states.

---

# Builder

Preview must always reflect actual public page behavior.

Builder preview and public page must stay synchronized.

---

# Profile

profile.png is the canonical source of truth.

If a design conflict occurs:

profile.png wins.

---

# Accessibility

All buttons:

44px minimum touch target.

All forms:

Proper labels.

All images:

Alt text.

---

# Principle

Clarity over decoration.

Function before aesthetics.

Consistency before creativity.
