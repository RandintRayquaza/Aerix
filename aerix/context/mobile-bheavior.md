# mobile-behavior.md

## Philosophy

Aerix must be fully usable on mobile devices.

Users should never need a desktop computer.

Everything available on desktop must be available on mobile.

---

# Navigation

Desktop:

Sidebar

Mobile:

Bottom Navigation

Items:

- Dashboard
- Builder
- Themes
- Analytics
- Library
- Settings

Domains accessible through Settings.

---

# Dashboard

Desktop:

Multi-column layout.

Mobile:

Single-column cards.

Cards stack vertically.

---

# Builder

Desktop:

3-column layout.

- Blocks
- Editor
- Preview

Mobile:

Tab navigation.

Tabs:

- Blocks
- Editor
- Preview

Only one tab visible at a time.

---

# Themes

Desktop:

Editor + Preview

Mobile:

Theme controls on top.

Live preview below.

---

# Analytics

Desktop:

Full dashboard layout.

Mobile:

Cards become stacked.

Charts become swipeable.

Tables become horizontal scroll containers.

---

# Media Library

Desktop:

Grid layout.

Mobile:

2-column layout.

Upload actions remain fixed.

---

# Settings

Desktop:

Multi-section layout.

Mobile:

Accordion sections.

---

# Domains

Desktop:

Card layout.

Mobile:

Single-column form layout.

---

# Uploads

All uploads supported on mobile.

Supported:

- Camera
- Gallery
- Files

---

# Preview

Desktop:

Side-by-side preview.

Mobile:

Dedicated Preview tab.

---

# Touch Targets

Minimum size:

44px

All buttons.

All icons.

All actions.

---

# Horizontal Content

Allowed:

- Gallery
- Instagram Reels
- YouTube Carousel

Must support touch gestures.

---

# Principle

Mobile is not a secondary experience.

Mobile and desktop must have feature parity.
